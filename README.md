# weather-app

Generated by `prisma-next init` with the Minimal template.

## First run

1. `bun run db:init` — applies your contract to the database (creates tables, signs the marker).
2. `bun run dev` — runs `src/index.ts`, which writes one row and reads it back.

### Required environment variables

- `DATABASE_URL` — Postgres connection string.
- `BETTER_AUTH_SECRET` — random secret used by BetterAuth to sign session cookies and tokens; generate one with `bun -e 'console.log(crypto.randomBytes(32).toString("base64"))'`. Put it in `.env` locally and configure it explicitly for Prisma Compute preview and production environments.
- `BETTER_AUTH_URL` — the public app origin (for example, the Prisma Compute deployment URL). Configure it per preview/production environment so BetterAuth's CSRF origin validation uses the external hostname rather than an internal proxy origin.

## Available scripts

- `bun run dev` — run the Prisma Next sample script

### Database and migrations

- `bun run contract:emit` — emit contract artifacts after contract changes
- `bun run db:init` — initialize database state manually
- `bun run db:update` — update database state manually
- `bun run migration:plan` — create a migration plan
- `bun run migrate` — apply a planned migration
- `bun run db:seed` — insert sample users manually (run after `db:init`)
