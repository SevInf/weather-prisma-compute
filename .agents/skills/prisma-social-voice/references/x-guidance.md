# X guidance

Use this for Prisma posts on X.

## What good looks like

- Lead with the practical point.
- Keep the post concise and readable in one pass.
- Use plain language.
- Make the reader care in the first sentence.
- Mention the product only when it clarifies the point.

## What to avoid

- Empty one-liners.
- Forced suspense.
- Broad "future of" claims.
- Feature piles that read like release notes.
- Over-polished corporate language.
- Vague teaser tags like `big things coming` without saying what area is changing.

## Better patterns

- State the user problem, then the product improvement.
- Use one proof point or one workflow detail.
- Prefer a plainspoken sentence over a slogan.
- If teasing future work, name the area and end with a complete sentence about where to follow updates.

## Example shift

Weak:

Prisma Postgres changes everything.

Better:

Prisma Postgres gives TypeScript teams a managed PostgreSQL workflow that fits the same Prisma setup they already use for schema and client generation.
